プログラミングII演習課題の手引き
4_1_pointer

1 説明

ポインタの練習課題である．実用的なものではない．

変数aとf，配列s[]にscanfで入力された値が入っている．これらの変数などのアドレスをポインタに代入しなさい．このポインタを使って，aの値をbに，fの値をgにコピーし，配列s[]の文字列を配列t[]にコピーしなさい．

ポインタの練習なので，b = a;やt[0] = s[0];のようにポインタを使わずに代入してはいけません．また，scanfとprintfの引数を書き換えないでください．これらの条件を満たさないプログラムはテストに成功しても合格点にはしません．

入力：標準入力に整数，実数，文字列が与えられる．

出力：入力された整数，実数，文字列．

コンパイル：

    $ gcc -Wall -o pointer pointer.c

#include <stdio.h>

int main() {
  int a, b = 0;
  float f, g = 0.0;
  char s[11], t[11] = "0123456789";
  int i;

  scanf("%d", &a);
  scanf("%f", &f);
  scanf("%10s", s);

  // ただし，a,f,sのアドレスをポインタに格納する．

  // ポインタを使ってa,f,s[]の値をb,g,t[]にコピーする．
  
  printf("%d\t%f\t%s\n", b, g, t);

  return 0;
}

4 ヒント

  • ポインタを使わずに次のように直接代入しても期待される出力をするプログラムになる．ただし，この課題はポインタの使い方を学ぶ課題なのでこのような直接代入はしてはならない．代入文の右辺にあるa，f，s[i]をポインタを使った書き方に書き換えること．

    scanf("%d", &a);
scanf("%f", &f);
scanf("%10s", s);

b = a; // 直接代入
g = f; // 直接代入
for (i = 0; i < 11; i++)
  t[i] = s[i]; // 直接代入

printf("%d\t%f\t%s\n", b, g, t);

  • 明快入門C 60 ポインタの基本機能，64 ポインタと配列，65 ポインタと文字列．

  • ポインタの演算子はアドレス演算子(&)と間接参照演算子(*)．アドレス演算子&は変数のアドレスを取得する．間接参照演算子*はポインタの指す変数を読み書きする．

    int y, x;
int *p;  // pはポインタだと宣言している．*pはint型変数

x = 1;
y = 2;
p = &x;  // 変数xのアドレスをpに代入
y = *p;  // アドレスpに格納されている変数の値をyに代入．代入後のyの値は1．
*p = 3;  // アドレスpに格納されている変数xに1を代入．代入後のxの値は3．

    変数はメモリの特定のアドレスに格納されている．変数が格納されているアドレスを取り出すのがアドレス演算子(&)である．他方，特定のアドレスに格納されている変数を読み書きするために使うのが間接参照演算子(*)．

  • 配列a[10]の先頭要素のアドレスをpに代入すると，配列要素s[1]を読み書きしたいときに，a[1]の代わりにp[1]や*(p + 1)が使える．

    int a[10];
int *p;

p = a; // aの先頭要素のアドレスをpに代入．
// p = &a[0]; とも書ける．
p[1] = 10;     // a[1]に10を代入する
*(p + 2) = 13; // a[2]に13を代入する
*p = 15;       // a[0]に15を代入する
p++;           // ポインタを次の要素a[1]に動かす．p = &a[1]と同じ．
*p = 16;       // a[1]に16を代入する
p += 2;           // ポインタを二つ後ろ要素a[3]に動かす．p = &a[3]と同じ．
*p = 16;       // a[2]に16を代入する
