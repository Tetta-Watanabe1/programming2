4.1 課題ファイルの準備（programming2ディレクトリ）

ここまでの演習準備でC:\cygwin64\home\(あなたのWindowsユーザー名)\programming2 ディレクトリができているはずです．プログラミングIIの課題はこのprogramming2ディレクトリ内に作成しましょう．

1週目の課題に関するファイルはMoodleの1.zipにあります．Moodleから 1.zipファイル（以降，ZIPファイルと書く）をダウンロードします．ダウンロードした1.zipファイルをprogramming2ディレクトリに移動します． Moodleから1.zipファイルをダウンロードする際に，次のように programming2ディレクトリに保存すると移動の手間がいらないのでおすすめです．

  1. Moodle上の1.zipを右クリックして「名前をつけてリンクを保存」を選ぶ1．
  2. C:\cygwin64\home\(あなたのWindowsユーザー名)\programming2 フォルダーを選んで「保存」する．

ZIPファイルの展開は，ターミナル（Cygwin64 Termial）で次のように unzipコマンドを実行するとできます．もちろん，unzipコマンドの代わりに，エクスプローラーで1.zipファイルを開きその中の1_1_hello_worldフォルダをprogramming2フォルダにコピーしても構いません．

4.3 プログラムの設計

それでは，hello_world.cファイルにプログラムの設計を書きましょう．まず，TeraPadなどのテキストエディタでhello_world.cを開きます．プログラムの大まかな設計をhello_world.c中の「入出力と動きの説明」部分に書きます．いきなりC言語で書くより慣れた日本語で書く方がこれから作成するプログラムを設計しやすいはずです．これらの説明は課題の途中で友人や教職員と相談するときや，後日自身でプログラムを読み直すときにも役に立ちます．

  • 入出力の説明：（どんな入出力のプログラムか）
  • 動きの説明：（どの制御文を使ってどのように制御するか．また，変数名ごとに役割を説明する）

動きの説明には，どのような構造のプログラムにするか，for文など繰り返し制御文をどう使うか，使う予定の変数名と役割などを書きましょう．無駄に複雑な例を次の節に示しますが，まねせずに自分で考えてください6．

レポート提出時にプログラムの内容と「入出力と動きの説明」が一致するように説明を整えてください．プログラムの作成中に制御文や変数，配列，配列要素数が増減したり，それらの役割が変わるかもしれません．mainの中を書き換えたときに「動きの説明」も書き換えておけば，書き換え忘れを防げます．

4.4 プログラムの作成

設計に沿ってプログラムを書きます．プログラムは次のようにコメント部分より 後ろの行 に書いてください．

/**
（略）

入出力の説明：Hello world!を4行出力する

動きの説明：
 for文Aでprintf文を4行のうちの2行分繰り返す．
 続いて，残り2行のうち1行のprintf文をwhile文Bで繰り返す．
 最後の1行はprintf文Cで出力する．
 変数iはfor文AとBの繰返し変数．

（略）
*/

#include <stdio.h>

int main() {
  int i;

  for (i = 0; i < 2; i++) { // for文A
    printf("Hello world!\n");
  }
  i = 1;
  while (i > 0) { // while文B
    i--;
    printf("Hello world!\n");
  }
  printf("Hello world!\n"); // printf文C
  return 0;
}

4.5 プログラムの整形

かっこに応じた字下げなどプログラムソースを読みやすく維持しましょう．字下げや改行位置が乱れてプログラムが読みにくくなったら，次のコマンドを使ってソースを整形しましょう7．

$ cd ~/programming2/1_1_hello_world
$ clang-format -i hello_world.c

または

$ cd ~/programming2/1_1_hello_world
$ make format

テキストエディタとして導入の容易なTeraPadを使っていますが，プログラムソースの整形を補助してくれる Visual Studio Code, Atom， Vim， Emacs などの様々なエディタがあます．例えば，Visual Studio Code だと Ctrl+K，Ctrl+Fでカーソル周辺を整形してくれます8．色々試して自分に合ったものを選びましょう9．

4.6 プログラムのコンパイル

ターミナル（Cygwin64 Termial）で次のコマンドを実行してコンパイルします10．

$ cd ~/programming2/1_1_hello_world
$ gcc -Wall -o hello_world hello_world.c

gccコマンドの最後の引数hello_world.cがコンパイルするファイル名です．他の引数-Wall -o hello_worldはgccに対するオプションです．-o hello_worldオプションによってコンパイラが生成する実行ファイル名が hello_world.exeになります11．

-Wallオプションはコンパイラが警告する項目を増やすオプションです．未使用の変数がある場合など警告してくれます．警告はエラーではないのでコンパイルは正常に終了します．とはいえ，警告されるプログラムは実行時に問題が起こる可能性がありますので，警告内容を理解しましょう 12．

4.7 プログラムの実行

コンパイルに成功したらプログラムを実行しましょう．自分が考えた通りに動作するか確認します．うまくいかないときは，授業時間中に教職員やTAに相談しましょう．

この課題では入力がないので出力は1通りですが，他の課題では様々な入力で試せるような課題になっています．入力の例として，2章の「テスト一覧」にプログラムを実行するときのコマンドラインと入力，それに応じた出力が示されています．少なくともこれらの入力に対して，期待通りの出力になることを確認しましょう．同じ出力にならなければ，同じにならない理由を考えて，プログラムを修正してください．多くの課題では複数のテストがあります．

$ cd ~/programming2/1_1_hello_world
$ ./hello_world
（略）

実行したときの出力がテスト1に書かれた出力と一致するか調べるコマンドがあります．次のようにdiffコマンドを使って出力を1行ずつ比較できます 13．コマンドライン末尾の1.correctの1はテストの番号ですので，必要に応じて書き換えてください．すべての行が一致する出力であれば次のように identicalと表示されます．

$ ./hello_world | diff -us - tests/1.correct
Files - and tests/1.correct are identical

出力に異なる行がある場合どの行が違っているかを次のように示してくれます． @@で始まる行以降の行頭の+や-が行の過不足を表しています．-の行が余分な出力で，+の行は足りない出力です．プログラムを修正して過不足をなくします．この例は，hello_worldプログラムが1行目にHello Hirodai!を出力し，残り4行はHello world!を出力する場合を示しています．正しい出力はHello world!が5行続くものとします．

$ ./hello_world | diff -us - tests/1.correct
--- -   yyyy-mm-dd HH:MM:SS
+++ tests/1.correct     yyyy-mm-dd HH:MM:SS
@@ -1,4 +1,4 @@
-Hello Hirodai!
+Hello world!
 Hello world!
 Hello world!

4.8 提出用テストの実行

「テスト一覧」に書かれたすべてのテストに成功するようにプログラムを修正してください．ターミナルで次のコマンドを実行するとテスト一覧のテストを実行できます14．

$ cd ~/programming2/1_1_hello_world
$ make test
テスト 1 成功（passed）
すべてのテストに成功しました．(1)

make testコマンドを実行して「すべてのテストに成功しました」と表示されればテスト成功です15．テストに成功したら，レポート作成と提出に進みましょう．成功しない場合は，プログラムの設計や作成のところ（4.3，4.4）に戻りましょう．

なお，hello_world-test-history.txtファイルにはテストするたびに，プログラムのソースファイルhello_world.cの変更箇所とテスト結果が格納されます16．

テストに失敗したときの例を示します．この例ではテスト 1 に失敗しています．ターミナルで./hello_worldを実行して出力を確認しましょう．正しい出力はこのファイル中の「テスト一覧」で確認しましょう．

$ make test
テスト 1 失敗（failed）
テストの内容は README_hello_world.html のテスト一覧を見なさい．
プログラムを確認してください．(0)

次のように表示されるのは，hello_world.cを書き換えた後にコンパイルしていない場合です．「プログラムのコンパイル」の節に戻りgccコマンドでhello_world.cをコンパイルし直しましょう．

$ make test
次のコマンドを実行して hello_world.c をコンパイルしてください．
gcc -Wall -o hello_world hello_world.c
make: *** [hello_world] Error 1

テストに失敗しているけど，どういう失敗なのかわからないときは，授業時間中に教職員やTAに相談しましょう．

  13. この講義の範囲ではないですが，この注釈でdiffコマンドの説明をします． diffコマンドは二つのファイルを行単位で比較するコマンドです．この例では-とtests/1.correctの二つのファイル名を指定しています．比較は行単位で行います．tests/1.correctは，この名前のテキストファイルがありますので探してください．-はファイルの代わりに標準入力を使うという意味で，少し特殊でファイルはありません．-は直前に./hello_world |と組み合わせて使います．bash（バッシュ）の記号|はパイプと呼び，|の左のコマンドを実行してその出力を右のコマンドの入力にするという役割を果たします．つまり./hello_world | diff - fileというコマンドラインを実行すると，diffコマンドの一つ目のファイルの代わりに./hello_worldの出力が使われ，fileという名前のファイルと1行ずつ比べて違いを出力します． diffコマンドの直後の-usは-uと-sの二つのオプションをまとめて指定しています．-uオプションは違いを出力する形式を指定するものです． -sオプションは一致する場合にidenticalと出力するオプションです．↩︎
