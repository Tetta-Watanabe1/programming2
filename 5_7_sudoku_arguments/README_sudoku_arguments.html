5_7_sudoku_arguments

1 説明

発展課題（未提出でも減点しない）

コマンドライン引数から数独の盤面を読み取り，盤面を出力せよ．

数独のルール：

  • タテ9列，ヨコ9行，3 \(\times\) 3の9つあるブロックのどれにも，1から9までの数字が一つずつ入るように空きマスに数字を入れる．

  • 詳しくは以下のページで確認してください．なお「数独(SUDOKU)」はニコリの登録商標です．

    http://nikoli.co.jp/ja/puzzles/sudoku/ (和文)

    http://nikoli.co.jp/en/puzzles/sudoku/ (英文)

  • 参考：数独の空きマスの最大数に関する論文．

    Gary McGuire, Bastian Tugemann, and Gilles Civario, “There is no 16-Clue Sudoku: Solving the Sudoku Minimum Number of Clues Problem via Hitting Set Enumeration,” Journal of Experimental Mathematics, vol. 23, no. 2, Taylor & Franscis, Aug. 2013. https://doi.org/10.1080/10586458.2013.870056 https://arxiv.org/pdf/1201.0749.pdf

コマンドライン引数：

  • 9 \(\times\) 9の盤面のそれぞれのマスを1文字で表す．

  • 引数は9個あり，それぞれの引数は9文字．

  • i番目の引数を盤面のi行に対応する．例：argv[1]が0行目． argv[2]が1行目．

    なお，盤面の行や列の番号を0から8で数える．配列添字が0から数えることに対応させている．一番上の行が0行目，一番下の行が8行目．一番左の列が 0列目，一番右の列が8列目（ほかの課題も同様）

  • i行目に対応する9文字のうち，j番目の文字は盤面のj列に対応する．例：argv[1][3]が0行3列のマスの文字．えている）

  • 引数の文字が0なら空きマス，1から9までの数字ならそのマスにその数字が入る．

入力：なし．

出力：

  • 前の課題と同じフォーマット．

  • 実行例のように3 \(\times\) 3マスの枠を+-|で囲んで，13行で表す．

  • 配列要素が0のマスは空白にする．

コンパイル：

    $ gcc -Wall -o sudoku_arguments sudoku_arguments.c

4 ヒント

  • コマンドライン引数と数独のマスの関係は次のようになる．

    • argv[1][0]は左上のマスに対応する数字であり， argv[1][8]は右上のマスに対応する数字である．
    • argv[9][0]は左下のマスに対応する数字であり， argv[9][8]は右下のマスに対応する数字であり，

  • これまでの課題のように配列に盤面を格納する必要はない．

  • これまでに作成したsudoku_print関数を使いたい場合は，int型の配列に盤面を格納する必要がある．一方，コマンドライン引数はchar型の配列である．コマンドライン引数の文字（例えばargv[1][0]）は，これまでに作成したsudoku_scanfと同様に，文字（つまり数字）であって整数値ではない．数字の'1'を整数の1に変換するには，'1'-'0'のように数字から'0'の文字コードを引く．逆に0から9の整数を数字に変換するには， 1+'0'のように'0'の文字コードを足す1．

  1. 10以上（二桁以上）の整数を数字列に変換するにはsprintf関数を使う方法がある．↩︎
