プログラミングII演習課題の手引き
3_3_mystrcpy

1 説明

配列src[]に格納された文字列を配列dst[]にコピーするvoid mystrcpy(char dst[], char src[])関数を書け．ただし，この関数内で strcpy関数などの文字列処理関数を使ってはならない．

main関数はmystrcpy関数の呼び出しのみ書き加えること．

mystrcpy関数の引数：

  • 第1引数：char型配列dst[] … コピー先の配列
  • 第2引数：char型配列src[] … コピー元の文字列が格納された配列

mystrcpy関数の戻り値：

なし．

プログラムの入力： 1行目の最初から一定長の文字列．残りは無視する．

プログラムの出力：入力された1行の文字列．

コンパイル：

    $ gcc -Wall -o mystrcpy mystrcpy.c

3 配布ファイル

ファイル名: mystrcpy.c

/**

氏名: 

学生番号: 

作成日: 

入出力の説明: 

動きの説明: 

感想: 

*/
#include <stdio.h>

/* ここにmystrcpy関数を書く．この関数内でSIZEや80を使う必要はない */

#define SIZE 80

int main() {
  char s1[SIZE];
  char s2[SIZE];
  char c;
  int i;

  // s1に1行読み込む．長くとも(SIZE-1)文字まで
  i = 0;
  c = getchar();
  while (c != '\n' && i < SIZE - 1) {
    s1[i] = c;
    i++;
    c = getchar();
  }
  s1[i] = '\0'; // 末尾のヌル文字

  // s2を数字で埋める（デバッグ用）
  for (i = 0; i < SIZE - 1; i++)
    s2[i] = '0' + (i % 10);

  // ここからmystrcpy関数を呼び出す

  printf("%s\n", s2); // コピーした文字列を出力する
  return 0;
}

4 ヒント

  • 文字列のコピーは1文字ずつコピーします．配列要素s[0]に格納された文字を別の配列d[]の3番目の要素にコピーするなら，次のように書きます．

    d[3] = s[0];

  • コピー元の文字列の長さはヌル文字'\0'を見つけることでわかります．

  • コピーした文字列の末尾にはヌル文字'\0'が必要です．つまり，配列 s[]に10文字の文字列を格納する場合は，s[0]からs[9]に文字列の各文字を格納します．さらに，次のように最後の文字の次の要素s[10]に ヌル文字 '\0'を代入します．

    s[10] = '\0';

  • 10文字の文字列を配列に格納するには，末尾のヌル文字を含めると11要素あるので，この配列の要素数（大きさ）は11以上必要です．

  • 配列の名前dstとsrcは目的地を意味するdestinationと情報源を意味するsourceの子音を抜き出した略語です．

  • 配布ファイル中のprintf関数の第二引数を次のようにs1にすれば mystrcpy関数を書かなくても正しい出力になりますが，文字列をコピーする関数を書く課題ですので，このような変更だけで提出してはいけません．

      printf("%s\n", s1); // 入力した文字列を出力する

  • （プログラミングIの復習）11要素の配列aはint a[11];のように宣言して，a[0]からa[10]までの11要素を読み書きできます．a[11]に読み書きすると他の変数の値が書き変わるなど，プログラムの挙動がわからなくなります．

    int a[11]; // 11要素の配列を宣言

a[0] = 'a';
a[10] = 'k';
a[11] = 'l'; // この行のように，宣言していない配列要素に代入してはならない．
  // 残念ながら，この代入はコンパイル時と実行時ともにエラーにはならず，
  // これ以降のプログラムがおかしな挙動をすることがある．
