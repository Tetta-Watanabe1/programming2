プログラミングII演習課題の手引き
3_2_ascii_code

1 説明

文字と整数nが入力されたら，入力された文字の文字コードを1行目に出力し， 2行目に文字コード表の並びに沿って入力された文字からn文字を出力するプログラムを書け．

入力：

  • 標準入力から1文字と整数nを読み込む．

出力：

  • 1行目に入力された文字のASCIIコード（10進表現の整数値）を出力する．

  • 2行目に入力された文字からn文字を出力する．行末は改行すること．

コンパイル：

    $ gcc -Wall -o ascii_code ascii_code.c

4 ヒント

  • 標準入力から1文字読み込むにはint getchar()関数を使うとよい．あるいはscanf("%c")を使ってもよい．（getcharについて詳しくは明快入門C 42 文字入出力1 基本用法やターミナルでman getcharを見よ）

  • 標準入力から整数nを読み込むにはscanf関数を， int n; scanf("%d", &n)のように使うとよい．（明快入門C 46 数値入出力1 基本用法）

  • 文字は数値として扱える．それぞれの文字に割り当てられた数値はASCIIコード表を見よ（明快入門C pp.244-245 の文字コード表）．aの次はb， bの次はcのように英文字は順に並んでいる．例えば，次のプログラムでは変数b1とb2は同じ文字bを表す．文字を'（シングルクオート）で囲んでいることに注意．文字列を囲む"（ダブルクオート）とは区別して使う．

    char b1, b2;
b1 = 'a' + 1;
b2 = 'b';

  • ASCIIコード表の文字と数値の対応を覚えるのは構わないが，上の例の代わりに次のように書くのはやめよう．文字定数'a'の代わりに数値97を書いても，プログラムが読みにくくなるだけである．

    b1 = 97 + 1; // 読みにくい書き方の例．'a'の代わりに97
b2 = 98; // 読みにくい書き方の例．'b' の代わりに98

    以下のようなprintf文を見かけたら，一度実行してシングルクオートで囲んだ文字定数に書き換えればよい．あるいはダブルクオートで囲んだ文字列定数にまとめた方がいいだろう．

    printf("%c%c%c%c%c%c%c%c%c%c%c%c%c",
       72, 101, 108, 108, 111, 32, 119, 
       111, 114, 108, 100, 33, 10);

  • char型変数の文字をprintf関数で出力する場合の書式指定は%cである．一方，同じ変数のASCIIコード（整数値）を出力する場合は%dを使う．（%dを指定した場合，char型からint型にキャスト（型変換）されて printf関数に渡されている）．

    char b;
b = 'A';
printf("%c %d\n", b, b); // A 65 と出力される
